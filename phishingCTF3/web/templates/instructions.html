Ethical Use Only – Do Not Enter Real Credentials

Scenario

A suspicious email was reported. Analyze the artifacts, decode tracking, observe credential capture behavior, and enumerate IOCs.

Tasks

  1. Review the phishing email (/email) – list 5 indicators.
  2. Extract spoofed vs. actual link destination.
  3. Decode base64 tracking token (tid parameter).
  4. Submit test credentials – observe behavior.
  5. Identify storage mechanism for captured credentials (view sanitized log at /logview).
  6. List two browser/user interface features that could alert a user.
  7. Build IOC JSON based on schema (/ioc-schema).
  8. Find OPSEC mistakes in source (download zip artifacts).
  9. Add defensive banner (already visible) – describe rationale.
  10. Summarize attempts using /scoreboard output.

Endpoints

  • / (phishing form – optionally append ?tid=TOKEN)
  • /email (rendered phishing email)
  • /download/email.eml (raw email)
  • /download/site.zip (artifact bundle)
  • /ioc-schema (JSON schema)
  • /scoreboard (JSON attempt stats)
  • /logview (sanitized captured attempts)

Ethics

No real credential use. No external mailing. Lab is isolated for defensive study only.
